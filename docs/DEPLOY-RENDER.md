# Deploying provinyl-backend to Render

The service runs as a Docker web service on Render, defined by `render.yaml`
(a [Blueprint](https://render.com/docs/blueprint-spec)). MongoDB is **not** on
Render ŌĆö use **MongoDB Atlas** and pass its URI as `MONGO_URI`.

## Prerequisites

1. **MongoDB Atlas** ŌĆö create a free M0 cluster, a DB user, and allow network
   access (`0.0.0.0/0` for Render, or Render's static egress IPs on paid plans).
   Copy the `mongodb+srv://ŌĆ”` connection string.
2. **Discogs app** ŌĆö register at <https://www.discogs.com/settings/developers> to
   get a **Consumer Key** and **Consumer Secret**. The callback URL is set after
   the first deploy (you need the Render URL first).
3. **A `TOKEN_ENC_KEY`** ŌĆö generate one:
   ```bash
   openssl rand -hex 32
   ```

## Deploy

1. Push to GitHub (already done ŌĆö `master`).
2. Render dashboard ŌåÆ **New ŌåÆ Blueprint** ŌåÆ connect `jayjonesdev/provinyl-backend`.
   Render reads `render.yaml` and creates the `provinyl-backend` web service.
3. When prompted, fill the `sync: false` env vars:
   | Key | Value |
   |---|---|
   | `MONGO_URI` | your Atlas `mongodb+srv://ŌĆ”` string |
   | `DISCOGS_CONSUMER_KEY` | from the Discogs app |
   | `DISCOGS_CONSUMER_SECRET` | from the Discogs app |
   | `CLIENT_ORIGIN` | the deployed `provinyl-web` origin (e.g. `https://provinyl.vercel.app`) |
   | `TOKEN_ENC_KEY` | the `openssl rand -hex 32` value |
   | `DISCOGS_CALLBACK_URL` | set in step 5 |

   `JWT_SECRET` is generated by Render; `NODE_ENV`, `PORT`, and the JWT expiries
   are already set.
4. Deploy. Render builds the Dockerfile and health-checks `/api/v1/health`. Note
   the service URL, e.g. `https://provinyl-backend.onrender.com`.
5. Set **`DISCOGS_CALLBACK_URL`** to
   `https://provinyl-backend.onrender.com/api/v1/auth/callback`, and set the same
   URL as the **Callback URL** in your Discogs app settings. Save ŌåÆ redeploy.

## Verify

```bash
curl https://provinyl-backend.onrender.com/api/v1/health
# {"status":"ok","timestamp":...}
```

Then open `https://provinyl-backend.onrender.com/api/v1/auth/login` from the SPA
to start the OAuth flow.

## Notes

- **Cookies / CORS.** The session uses `SameSite=Lax`, `Secure`, httpOnly
  cookies. That requires the frontend and API to be on the **same registrable
  domain** (e.g. `app.provinyl.com` + `api.provinyl.com`). If they're on
  different sites (e.g. `*.vercel.app` + `*.onrender.com`), switch the cookies to
  `SameSite=None; Secure` in `src/auth/cookies.ts` (and ensure HTTPS). The SPA
  must send requests with `credentials: 'include'`.
- **Free plan** spins the service down when idle; the first request after idle is
  slow. The in-memory cache, rate-limit semaphore, and pending-OAuth store are
  per-instance ŌĆö fine for a single free/starter instance, but move them to Redis
  before scaling to multiple instances.
- **CI** (`.github/workflows/ci.yml`) runs lint/typecheck/test/build on every
  push; Render auto-deploys `master` on success.
